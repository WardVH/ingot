# temporal-viz — the golden-record story & time machine

An interactive presentation of the golden-record way of working. The front page (`/`) is a
**guided story** in six chapters — why destructive merging fails, claims-not-records, priority
resolution, the as-of time machine, the over-merge guard, and the mistake-is-cheap finale (wrong
merge → contradiction surfaces → split → everything re-homes). The standalone scenes remain at
**`/machine`** (drag a date, watch medipim entity **422156** become resolvable) and **`/guard`**.

Astro + React islands, animated with [motion](https://motion.dev). It reimplements **none** of the
engine — every engine chapter replays precomputed snapshots the real Elixir engine produces.

## Run

```bash
cd viz
npm install
npm run dev          # http://localhost:4321
```

Other scripts: `npm run build` (static site → `dist/`), `npm run preview`, `npm test` (Vitest).

## Presenting

- `→` / `←` (or the edge buttons) advance step by step; `1`–`6` jump straight to a chapter.
- Every step has a one-line on-screen caption, so the story also works self-driven as a link.
- Deep-link any position with `?c=<chapter>&s=<step>` (1-based), e.g. `/?c=6&s=3` opens the
  mistake arc at the contradiction beat — refresh-safe mid-talk.

## Regenerate the data

The browser can't run Elixir, so the data is exported once from the engine. From the **repo root**:

```bash
mix run temporal_export.exs        # writes viz/src/data/temporal.json  (chapters 4–5)
mix run demo_export.exs            # writes viz/src/data/story.json     (chapters 1–3, 6)
```

Both files are committed (so the site builds without Elixir) **and** regenerable. They're
generated — don't hand-edit them; re-run the exports instead. `test/demo_export_test.exs` replays
the story scenarios through the engine, so the demo can't silently drift from engine behaviour;
the vitest schema test pins the JSON shape the components expect.

## Layout

```
viz/
  src/
    data/temporal.json     # generated by temporal_export.exs (time machine + guard)
    data/story.json        # generated by demo_export.exs (the story chapters)
    lib/types.ts           # temporal.json shape
    lib/story.ts           # story.json shape
    lib/snap.ts            # scrubber snapping (+ snap.test.ts)
    lib/steps.ts           # story navigation (+ steps.test.ts incl. story.json schema)
    lib/narration.ts       # one-line caption per (chapter, step)
    components/
      Story.tsx            # the guided-story island (rail, keyboard nav, captions)
      chapters/            # OldWay · ClaimsIntro · PriorityDuel · MistakeArc · shared stage
      TimeMachine.tsx      # scene — also embeds as chapter 4 (`embedded`)
      OverMergeGuard.tsx   # scene — also embeds as chapter 5 (`embedded`)
    pages/                 # index (story) · machine · guard
    styles/global.css
```
