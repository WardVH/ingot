# Ingest fixtures

## `medipim_be_422156.*` тАФ real legacy entity 422156

The full delta history of one real medipim product (legacy `entity` 422156, an Aderma Primalba
wash gel), used as the PoC fixture for the ingest pipeline. It is **real data**, not synthetic.

| file | what |
|------|------|
| `medipim_be_422156.raw.jsonl` | **Ground truth.** 1676 real `products_deltas` rows, one JSON object per line, time-ordered, undecoded. Exactly as stored in medipim (`[opcode, key, value]` event triples). No decode decisions baked in. |
| `gen.exs` | The decoder/**oracle** (general over markets): applies contract-C decode rules to a raw dump and emits the envelope. One-off fixture tooling тАФ **not** the runtime ingest (see below). Identity classification is driven by `CodeRegistry.identity_fields/0`, so it must run via `mix run` (lib/ compiled), not bare `elixir`. |
| `gen_422156.exs` | A thin, zero-argument wrapper that pins `gen.exs` to the Belgian fixture тАФ the byte-for-byte regression guard. |
| `medipim_be_422156.json` | The decoded-but-unresolved **`HistoryEnvelope`** (contract C). Generated from the raw dump by the oracle. This is what the ingest pipeline consumes and what medipim's future PHP endpoint (bead gr-867) must reproduce. |

Regenerate the envelope after editing decode rules (via `mix run`, so `CodeRegistry` is loadable):

```bash
mix run test/ingest/fixtures/gen.exs medipim-be 422156   # or the pinned wrapper:
mix run test/ingest/fixtures/gen_422156.exs
```

### Provenance

Exported 2026-06-08 from a production dump (`medipim_db_08-06-2026...csv`) loaded into the local
`medipim2-mysql` Docker container (`medipim_test.products_deltas`, FK checks off тАФ only the deltas
were needed). The raw JSONL is a faithful re-export of those 1676 rows as utf8mb4.

### Why a one-off oracle and not the production loader

Contract C keeps medipim's decode (`ProductDeltaApplier`/meta) on medipim's side of the wall; the
production system-of-record ingest consumes envelopes emitted by medipim's own PHP endpoint
(gr-867), reusing battle-tested code. `gen_422156.exs` exists **only** to bootstrap this committed
fixture from a one-time dump. Its output *is* the contract the endpoint must match тАФ so when gr-867
is built, diff its emitted envelope for 422156 against `medipim_be_422156.json` to validate.

### Shape of this fixture (decoded)

- **930 events** kept тАФ 23 identity, 127 attribute, 12 edge, 768 media.
- **819 touch-only deltas dropped** (`dropped_meta_count`); `last_touched_at` preserved.
- **Media churn dominates** (тЙИ82% of events are `media` add/remove of asset ids). It is faithful
  to the log but pure noise for identity re-derivation; downstream clustering ignores it.

### The identity story (why 422156 is a good fixture)

All three source orgs (1034, 1035, 44) converge on **CNK `3612173`** and canonical GTIN
`03282770146004`, but **not at the same time** тАФ org 44 began divergent (a different EAN
`3282770049374`, no CNK) and only converged its EAN in 2023-08 and its CNK in 2024-03. A snapshot
fold sees one clean cluster ("legacy was right"); the temporal pass recovers *when* org 44 merged
in. The history also exercises the GTIN scheme migration (`ean`тЖТ`eanGtin13`тЖТ`eanGtin14`), an op-4
delete, and 2026 set-null cleanups.

### Classification decisions (see `gen.exs` for the decode rules; `CodeRegistry` for the identity set)

- `legacyId` (medipim's own previous-system id) тЖТ dropped as meta; it is not a product code.
- `organizations` тЖТ `edge` (structural; derivable from per-source events, may be ignored downstream).
- `update_sources` opcode тЖТ dropped (a survivorship recompute, not a data change).

## `medipim_fr_347025.*` тАФ real legacy entity 347025 (the first French extract)

The full delta history of one real **French** medipim product (legacy `entity` 347025, a Biotherm
face-care product, cbId 52549). The first non-Belgian fixture тАФ it carries the FR identity schemes
(`cipOrAcl7` тЖТ `:cip_acl7`, `acl13`, and the EAN/`eanGtin13`/`eanGtin14` family) and **no** CNK,
exercising the gr-6k4 registry end-to-end.

| file | what |
|------|------|
| `medipim_fr_347025.raw.jsonl` | **Ground truth.** 69 real `products_deltas` rows, one JSON object per line, time-ordered, undecoded. Converted from a CSV export by `csv_to_raw_jsonl.exs`. |
| `medipim_fr_347025.json` | The decoded `HistoryEnvelope`. Generated by `mix run test/ingest/fixtures/gen.exs medipim-fr 347025`. |
| `csv_to_raw_jsonl.exs` | One-off CSV тЖТ raw.jsonl converter (stdlib only): peels the front `id,entity` and the tail `"tag",created_at,created_by` off each row positionally, leaving the CSV-quoted `events` JSON array (un-double `""` тЖТ `"`, then decode). |

Regenerate:

```bash
mix run test/ingest/fixtures/csv_to_raw_jsonl.exs 347025 <export.csv> test/ingest/fixtures/medipim_fr_347025.raw.jsonl
mix run test/ingest/fixtures/gen.exs medipim-fr 347025
```

### Shape of this fixture (decoded)

- **114 events** kept тАФ 32 identity, 73 attribute, 9 edge, 0 media.
- **8 touch-only deltas dropped** (`dropped_meta_count`); `last_touched_at` preserved.

### The identity story (why 347025 is a good FR fixture)

Across 5 source orgs the history is busy тАФ set/add/remove plus many op-4 (source-in-value) deletes,
and the `eanGtin13_`/`eanGtin14_` value-prefix migration тАФ and two orgs end fully delisted. Yet every
surviving listing shares `cipOrAcl7` `4440813` and/or `acl13` `3401344408137`, so identity re-derives
to **one** cluster / **one** surrogate key / **one** golden product, whose codes are `:cip_acl7`
`4440813`, `:acl13` `3401344408137`, and canonical GTIN `03367729203738` (the EAN `3367729203738`).
It is the single-entity `:stable` analog of 422156. (Cross-entity over-merge is gr-ose's concern.)
