1 · Ingestion

Substrate + Codes — claims, not truths

Sources never write the golden record. They assert claims, and every code is canonicalized on the way in: the GTIN family (EAN-8 / UPC-12 / EAN-13 / GTIN-14) collapses to one 14-digit form, national short codes are zero-padded — so equal trade items compare equal.

Canonicalization Codes

{:ean, "5410013112345"} → {:gtin, "05410013112345"}
Plus mod-10 validity, packaging-level indicator digit, and restricted-prefix detection.

Five kinds of claim Substrate.claim/5

identity codes that co-name a variant grouping code → product attribute code · field · value media asset → target code member_of code → collection

Each claim has two clocks: valid_from (true in the world) and recorded_at (learned by us). Per (source, slot) only the latest claim is live.

2 · Identity Resolution

Cluster → IdentityLedger — who is the same thing?

Clustering: connected components over each source's identity-code sets — two sets that share a code are the same variant. Except shared codes (steward-declared, e.g. one GTIN on a bundle and its unit): they ride along on every variant that bears them but never bridge clusters.

Reconciliation (IdentityLedger.decide): each new cluster is matched against established surrogate keys —

overlaps 0 keys mint SK_n

→ IdentityMinted

overlaps 1 key extend it

→ IdentityMembersChanged

overlaps ≥ 2 keys ⛔ gated — propose, never auto-merge

→ ConflictFlagged {:merge, keys} for a steward

1 key, ≥ 2 clusters split it

The GTIN-bearing / most-overlapping cluster keeps the key; the rest get fresh keys. → IdentitySplit

3 · Stewardship — the human loop

Stewardship — flag what the machine can't settle

Detected automatically

  • Attribute ties — top-priority sources disagree on a field → {:attr, key, field}
  • Code collisions — one variant's codes point at >1 product → {:collision, key}
  • Merge proposals — from the gated reconcile branch → {:merge, keys}

Steward verdicts

  • pick a value · approve / reject a merge · resolve a collision
  • mark shared — "two products legitimately use this code" → clean end-state, no more bridging
  • split — carve codes out of a key into fresh keys
every verdict is a ConflictResolved event, appended back to the log  ↺  — so the next fold (and the next reconcile) honors it

4 · Read Side — every projection is a fold

Four views, one log

Catalog Catalog · Survivorship · Priority

The golden record now: per key, fields decided by source-priority tiers per dimension; steward picks override. Media and categories attach by code, so they re-home automatically across splits and merges.

History History

Bitemporal time travel: project as-known-on × effective-on by filtering the two clocks before folding. Plus lineage — the full identity story of any key.

Customer API Api

Customers address by code, not key. A stale key still answers — with :merged → survivor or :split → parts redirects — and a change feed of identity events repairs local copies.

Public IDs PublicId

Identity-grade schemes like CNK: one canonical value per key (chosen by priority), the rest become aliases; an invariant check guarantees no code ever owns two keys.
