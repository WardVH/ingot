what Ingot does with it, step by step
Step 1

Write everything down — like a notary

No system edits the record directly. Every statement lands in a permanent logbook — who said what, when. Nothing is ever erased; corrections sit next to the mistakes they fix.

first, make the codes comparable
Step 2

Tidy the barcodes

The same barcode arrives spelled differently — like +32 478… and 0478… are the same phone number. Ingot rewrites every code into one standard spelling, and catches typos on the way.

now matching is possible
Step 3

Match what belongs together

Feeds that point at the same code are talking about the same product — no fuzzy guessing on names, so every match is explainable.

machines stop where judgment starts
Step 4

People decide the close calls

When the data honestly disagrees, the question goes to a review queue with the evidence laid out — and the verdict is written into the same logbook.
